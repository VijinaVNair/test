C: main() function▶

In C, the main() function is the starting point of any program. Every C program must have a main() function, otherwise the program won’t run. It is the function where execution begins. It can return an integer value, usually 0, which indicates that the program executed successfully.

Example:

#include <stdio.h>
int main() {
    printf("Hello World\n");
    return 0;
}

Here, the program starts from main(), prints "Hello World", and returns 0.

Sometimes main() can also accept arguments like int argc and char *argv[]. These are used to take command-line inputs.

Key points to mention in an interview:

  • main() is mandatory.
  • Execution always starts here.
  • Can return int or sometimes void (though int is standard).
  • Can accept command-line arguments.
C: Pointers▶

A pointer in C is a variable that stores the memory address of another variable.

  • Normal variable: stores value
  • Pointer variable: stores address (location of the value)

For example, if I have int a = 10;, I can declare a pointer as int *ptr = &a;.

Here, ptr stores the address of a.

I can access the value of a using the pointer by dereferencing it: *ptr gives 10.

Pointers are powerful because they let us:

  • Directly manipulate memory.
  • Pass large data structures to functions efficiently.
  • Handle arrays dynamically.

We can also do pointer arithmetic. For example, if ptr points to an array element, ptr + 1 points to the next element.

Example:

int arr[3] = {10, 20, 30};
int *ptr = arr;
printf("%d", *(ptr + 1)); // Prints 20

Pointers are essential in C because they provide flexibility and efficiency.

C: Dynamic Memory Allocation▶

Normally, memory is allocated at compile time (known as static allocation). But sometimes we don't know how much memory we'll need, for example, when reading user input or creating flexible data structures. Dynamic Memory Allocation (DMA) lets us allocate memory at runtime using functions from the <stdlib.h> library.

There are four main functions used for this: malloc(), calloc(), realloc(), and free(). I’ll explain each one with examples.

1. malloc() – Memory Allocation

malloc() stands for memory allocation. It allocates a single block of memory of a given size in bytes. The memory is not initialized, so it may contain garbage values.

Syntax: ptr = (castType*) malloc(sizeInBytes);

Example:

int *ptr;
ptr = (int*) malloc(5 * sizeof(int)); // Allocate memory for 5 integers

Here, ptr points to the first element of the memory block. We should always check if malloc() succeeded:

if(ptr == NULL) {
    printf("Memory allocation failed");
}

2. calloc() – Contiguous Allocation

calloc() stands for contiguous allocation. It allocates memory for multiple elements and initializes all bits to zero.

Syntax: ptr = (castType*) calloc(numElements, sizeOfElement);

Example:

int *ptr;
ptr = (int*) calloc(5, sizeof(int)); // 5 integers, all initialized to 0

This is useful when we want clean, initialized memory instead of garbage values.

3. realloc() – Reallocation

realloc() is used to resize previously allocated memory. This is useful if we initially allocated memory but later need more or less.

Syntax: ptr = (castType*) realloc(ptr, newSizeInBytes);

Example:

ptr = (int*) realloc(ptr, 10 * sizeof(int)); // Resize memory to hold 10 integers

If reallocation fails, realloc() returns NULL. So it’s good to assign it to a temporary pointer to avoid losing the original memory.

4. free() – Deallocate Memory

free() is used to release memory that was previously allocated by malloc(), calloc(), or realloc().

Syntax:

free(ptr);
ptr = NULL; // Avoid dangling pointer

It is essential to free memory to prevent memory leaks, especially in large programs or long-running applications.

Example Using All Functions

#include <stdio.h>
#include <stdlib.h>
int main() {
    int *ptr;
    // malloc
    ptr = (int*) malloc(5 * sizeof(int));
    for(int i=0; i<5; i++) ptr[i] = i+1;
    
    // calloc
    int *ptr2 = (int*) calloc(5, sizeof(int));
    
    // realloc
    ptr = (int*) realloc(ptr, 10 * sizeof(int));
    
    // free
    free(ptr);
    free(ptr2);
    ptr = NULL;
    ptr2 = NULL;
    
    return 0;
}
C: Static Function▶

In C, a static function is a function whose scope is limited to the file in which it is declared. That means it cannot be accessed from other files, even if those files are part of the same project.

The main use of static functions is encapsulation. It helps hide internal logic and prevents other files from using or interfering with it. This also avoids name conflicts if multiple files have functions with the same name.

A static function retains its lifetime throughout the program execution, just like a global variable. But unlike global functions, it is not visible outside its file.

Example:

#include <stdio.h>
static void greet() {
    printf("Hello from static function!\n");
}
int main() {
    greet(); // Works fine
    return 0;
}

If I try to call greet() from another file, it will not work because it is static.

So, to summarize, a static function is file-private, cannot be accessed externally, and is useful for hiding internal helper functions inside a file.

C: Local Variable▶

A local variable is a variable that is declared inside a function or a block. It only exists while the function is running. When the function ends, the variable is destroyed and its memory is freed.

Example:

void func() {
    int x = 10; // Local variable
    printf("%d", x);
}

Here, x exists only inside func() and cannot be accessed outside it.

Local variables are useful because they keep data private to the function and prevent unwanted changes from other parts of the program.

C: Global Variable▶

A global variable is declared outside all functions, usually at the top of the program. It can be accessed by any function in the same file.

If we use the extern keyword, it can even be accessed from other files.

Example:

#include <stdio.h>
int count = 0; // Global variable
void increment() {
    count++;
}
int main() {
    increment();
    printf("%d", count); // Prints 1
    return 0;
}

Global variables are useful for sharing data between functions, but they should be used carefully because any function can change them, which may cause bugs.

C: Control Statements▶

In C, control statements help us control the flow of the program. There are four main ones: break, continue, goto, and return.

1. break

break is used to exit from a loop or switch immediately.

Example:

for(int i = 1; i <= 5; i++) {
    if(i == 3) break;
    printf("%d ", i);
}
// Output: 1 2

Here, the loop stops when i becomes 3.

2. continue

continue skips the current iteration and goes to the next iteration of the loop.

Example:

for(int i = 1; i <= 5; i++) {
    if(i == 3) continue;
    printf("%d ", i);
}
// Output: 1 2 4 5

Here, 3 is skipped, but the loop continues.

3. goto

goto jumps to a labeled part of the program.

Example:

int i = 1;
label:
    printf("%d ", i);
    i++;
    if(i <= 3) goto label;
// Output: 1 2 3

goto can make code messy, so it’s usually avoided in structured programming.

4. return

return exits from a function and optionally returns a value.

Example:

int sum(int a, int b) {
    return a + b;
}
int main() {
    printf("%d", sum(2,3)); // Output: 5
    return 0;
}

It is used to send values back to the caller and end function execution.

C: Tokens▶

In C, tokens are the smallest meaningful units of a program. There are six types: keywords, identifiers, constants, strings, operators, and punctuation symbols.

  • Keywords are reserved words like int or return.
  • Identifiers are names we give to variables or functions, like sum or main.
  • Constants are fixed values like 10 or 'A'.
  • Strings are characters inside double quotes like "Viji".
  • Operators perform operations, for example +, -, or ==.
  • Punctuation symbols like ;, {}, or () help organize the code.
C: Predefined Functions▶

In C, a predefined function, also called a library function, is a function that is already provided by C in its standard library. We don’t need to write the code for it; we just call it to perform a specific task. These functions are declared in header files like stdio.h, math.h, string.h, etc. Using predefined functions makes programming faster and easier.

There are different categories:

1. Input/Output Functions

  • Declared in stdio.h.
  • Examples: printf(), scanf().
  • Use: Print or read data.

2. Mathematical Functions

  • Declared in math.h.
  • Examples: sqrt(), pow(), sin(), cos().
  • Use: Perform mathematical calculations.

3. String Handling Functions

  • Declared in string.h.
  • Examples: strlen(), strcpy(), strcmp().
  • Use: Work with strings like copying, comparing, or finding length.

4. Character Handling Functions

  • Declared in ctype.h.
  • Examples: isdigit(), isalpha(), toupper().
  • Use: Check or convert characters.

5. Time and Date Functions

  • Declared in time.h.
  • Examples: time(), clock(), difftime().
  • Use: Work with current time or measure program execution.

6. Other Utility Functions

  • Declared in stdlib.h.
  • Examples: malloc(), calloc(), exit().
  • Use: Dynamic memory allocation and process control.
C: Header Files▶

In C, header files contain predefined library functions. We can use these functions in our program by including the header file using #include. For example, #include <stdio.h> lets us use printf() and scanf(). Header files help reduce the amount of code we need to write and make the program simpler and less complex. So basically, whenever we want to use standard functions like input/output, math calculations, string operations, or memory management, we just include the relevant header file instead of writing the function ourselves.

C: Arrays▶

In C, an array is a collection of elements of the same data type, stored in contiguous memory locations. We use arrays when we want to store multiple values of the same type together.

Syntax: data_type array_name[size];

Example: int mark[5]; // Array of 5 integers

We access array elements using index numbers starting from 0. For example, mark[0] is the first element and mark[4] is the fifth element.

Types of Arrays

1. One-Dimensional (1-D) Array

A linear list of elements.

Example: int arr[5]; // 1-D array

2. Two-Dimensional (2-D) Array

An array of arrays, like a matrix.

Example: int arr[3][4]; // 3 rows, 4 columns

3. Three-Dimensional (3-D) Array

More than 2-D, used for multilayered data.

Example: int arr[2][3][4]; // 2 blocks, 3 rows, 4 columns

C: const Keyword▶

In C, the const keyword is used to declare a constant variable, which cannot be changed after it is initialized.

Example: const int x = 10;

Here, x is a constant, so if we try to assign a new value to x, the compiler will give an error. Using const helps prevent accidental changes to variables that should remain constant, making the program safer and more reliable.

C: Memory Leak▶

In C, a memory leak occurs when a program allocates memory dynamically but fails to free it after use. The memory becomes unreachable, but it still occupies space in RAM. Over time, this can cause the program or even the system to slow down or crash.

Cause: Repeatedly allocating memory using functions like malloc() or calloc() without freeing the old memory.

Solution: Always use free() to release dynamically allocated memory once it is no longer needed.

Example:

int *ptr = (int*) malloc(sizeof(int));
// use the memory
free(ptr); // frees the memory to prevent memory leak
C: Call by Value vs Call by Reference▶

In C, there are two ways to pass arguments to a function: call by value and call by reference.

1. Call by Value

  • The function receives a copy of the variable.
  • Any changes inside the function do not affect the original variable.
  • Safer because the original data cannot be changed, but it uses extra memory for the copy.

Example:

void fun(int x) {
    x = x + 10;
}
int main() {
    int a = 5;
    fun(a);
    printf("%d", a); // Prints 5
}

Here, a remains 5 outside the function because only a copy was modified.

2. Call by Reference

  • The function receives the address of the variable.
  • Any changes inside the function affect the original variable.

Example:

void fun(int *x) {
    *x = *x + 10;
}
int main() {
    int a = 5;
    fun(&a);
    printf("%d", a); // Prints 15
}

Here, a becomes 15 because the function modified the original variable through its address.

C: Compiler vs Interpreter▶

In C programming, there are two types of program translators: compiler and interpreter.

1. Compiler

  • The compiler compiles the entire code at once.
  • It displays all errors together after compilation.
  • Programs compiled by a compiler generally have faster execution.
  • Example: gcc for C.

2. Interpreter

  • The interpreter executes the program line by line.
  • It shows output or errors for each line during execution.
  • Programs executed by an interpreter generally have slower execution.
  • Example: Python interpreter, JavaScript engine.
C: Entry Loop vs Exit Loop▶

In C, loops are used to execute a block of statements repeatedly. There are two types of loops based on when the condition is checked:

1. Entry Loop

  • In an entry loop, the condition is checked first.
  • The statements inside the loop execute only if the condition is true.
  • Example: while loop
int i = 1;
while(i <= 3) {
    printf("%d ", i);
    i++;
}
// Output: 1 2 3

Here, the condition i <= 3 is checked before the loop executes.

2. Exit Loop

  • In an exit loop, the statements are executed first, and then the condition is checked.
  • It is also called an exit-controlled loop.
  • Example: do-while loop
int i = 1;
do {
    printf("%d ", i);
    i++;
} while(i <= 3);
// Output: 1 2 3

Here, the statements execute at least once, even if the condition is false.

C: Typecasting▶

In C, typecasting is the process of converting a variable from one data type to another. We use it when we want to temporarily change the type of a variable, usually for calculations or compatibility.

Types of Typecasting

1. Implicit Typecasting (Type Conversion)

  • Done automatically by the compiler.
  • Usually occurs when a smaller type is converted to a larger type.

Example:

int a = 10;
float b = a; // int automatically converted to float

2. Explicit Typecasting (Type Conversion)

  • Done manually by the programmer using parentheses.

Example:

float a = 9.8;
int b = (int) a; // float explicitly converted to int

C: Structures▶

DBMS: Basics & Applications▶

DBMS stands for Database Management System. It is a software that stores, manages, and organizes data in a structured way. A DBMS allows users to create, read, update, and delete data efficiently, while also ensuring data security and integrity.

Applications of DBMS

DBMS is widely used in many areas, such as:

  • Banking Systems – For managing accounts, transactions, and customer data.
  • Airline Reservation Systems – For booking tickets and managing flights.
  • Universities – To maintain student records, courses, and exam results.
  • Hospitals – For patient records, billing, and appointments.
  • E-commerce – To manage product catalog, orders, and inventory.
  • Library Management – To keep track of books, borrowing, and returns.
DBMS: Primary Key▶

In DBMS, a Primary Key is a column or a set of columns in a table that uniquely identifies each record. A primary key cannot have NULL values, and no two rows can have the same primary key.

Example:

CREATE TABLE Student (
    RollNo INT PRIMARY KEY,
    Name VARCHAR(50),
    Age INT
);

Here, RollNo is the primary key, so each student has a unique roll number.

DBMS: NULL vs UNIQUE Constraints▶

In a database, constraints are rules applied on columns to ensure data integrity.

1. NULL Constraint

  • Determines whether a column can accept NULL values.
  • Use NOT NULL to make a column mandatory.

Example: Name VARCHAR(50) NOT NULL
Here, the Name column cannot be left empty.

2. UNIQUE Constraint

  • Ensures that all values in a column are distinct.
  • Most DBMS allow one NULL value in a UNIQUE column.

Example: Email VARCHAR(50) UNIQUE
Here, no two rows can have the same email address.

DBMS: Degree of Relation▶

In DBMS, the degree of a relation is the number of attributes or columns in a table. It tells us how many fields are used to describe each record in the table.

Example:

CREATE TABLE Student (
    RollNo INT,
    Name VARCHAR(50),
    Age INT,
    Marks INT
);

Here, the Student table has 4 columns: RollNo, Name, Age, Marks. So, the degree of this relation is 4.

Important points:

  • Degree depends on the number of columns, not the number of rows.
  • It is useful in database design and normalization.
DBMS: Logical vs Physical Database▶

In DBMS, there are two ways to view a database: logical and physical.

1. Logical Database

  • Shows what data is stored and how it is related conceptually.
  • Focuses on structure, relationships, and constraints, like tables, columns, and ER models.
  • It is a high-level, user-facing view.

Example: A Student table with columns RollNo, Name, and Marks. Here, we are only concerned with what data exists and how it relates, not how it is stored.

2. Physical Database

  • Shows how and where data is actually stored on disk or in memory.
  • Focuses on storage, indexing, file organization, and access paths, such as data files, indexes, and memory blocks.
  • It is a low-level, implementation view.

Example: How the Student table records are stored on disk, the layout of files, and indexing methods for faster access.

DBMS: Database Languages (DDL, DML, DQL, DCL, TCL)▶

In DBMS, database languages are used to create, manage, and manipulate databases. The main types are: DDL, DML, DQL, DCL, and TCL.

1. DDL – Data Definition Language

Used to define the structure of the database.

Examples:

CREATE TABLE Student (...);
ALTER TABLE Student ADD Address VARCHAR(50);

2. DML – Data Manipulation Language

Used to modify data in the database.

Examples:

INSERT INTO Student VALUES (1, 'Viji', 95);

3. DQL – Data Query Language

Used to fetch or retrieve data.

Example:

4. DCL – Data Control Language

Used to control access to data.

Examples:

5. TCL – Transaction Control Language

Used to manage transactions.

Examples:

COMMIT;
ROLLBACK;
SAVEPOINT sp1;
DBMS: WHERE vs HAVING▶

In SQL, both WHERE and HAVING are used to filter records, but they work at different stages:

1. WHERE Clause

  • Filters individual rows before grouping.
  • Cannot use aggregate functions like SUM, AVG, COUNT.
  • Appears before GROUP BY in a query.

Example:

SELECT * FROM Sales
WHERE Amount > 1000;

This filters all sales greater than 1000.

2. HAVING Clause

  • Filters groups of rows after grouping.
  • Can use aggregate functions.
  • Appears after GROUP BY in a query.

Example:

SELECT Seller, SUM(Amount)
FROM Sales
GROUP BY Seller
HAVING SUM(Amount) > 5000;

This filters sellers whose total sales are above 5000.

DBMS: Transactions & ACID Properties▶

In DBMS, a transaction is a sequence of operations performed as a single logical unit of work. The key idea is: either all operations succeed or none are applied, which maintains database consistency.

Example of a Transaction

Consider a money transfer from Account A to Account B:

BEGIN;
UPDATE Account SET Balance = Balance - 1000 WHERE AccNo = 'A';
UPDATE Account SET Balance = Balance + 1000 WHERE AccNo = 'B';
COMMIT;

If any step fails, ROLLBACK undoes all changes.

Transaction control commands include:

  • COMMIT – save changes
  • ROLLBACK – undo changes
  • SAVEPOINT – set a point to roll back to

ACID Properties

Transactions follow ACID properties to ensure reliability:

  • Atomicity – All or nothing. A transaction must complete fully or not at all.
    Example: If debiting A succeeds but crediting B fails, the debit is undone.
  • Consistency – Data must remain valid. Database rules and constraints are preserved.
    Example: Total balance across accounts remains the same before and after transfer.
  • Isolation – Transactions do not interfere with each other. Each runs as if it is the only one.
    Example: Two users cannot book the same seat at the same time.
  • Durability – Once committed, changes are permanent. Survive system failures.
    Example: After a successful transfer, the records are preserved even if the system restarts.
DBMS: ER (Entity-Relationship) Model▶

Definition: "An ER model is a way to show how data is structured in a database. It uses entities, attributes, and relationships to give a high-level conceptual view of the data."

1. Entity

What it is: A real-world object that can be identified uniquely.

Example: A Student in a college, a Car, or a Book.

Types of entities:

  • Strong entity: Has its own primary key and does not depend on other entities.
    Example: Student with StudentID.
  • Weak entity: Cannot be identified alone; depends on a strong entity. Identified using a partial key + owner’s key.
    Example: Dependent of an Employee (like their child). It depends on EmployeeID + DependentName.

2. Attribute

What it is: A property that describes an entity.

Types with examples:

  • Key attribute: Uniquely identifies an entity. Example: StudentID for Student.
  • Composite attribute: Can be divided into smaller parts. Example: Name → FirstName + LastName
  • Multivalued attribute: Can have multiple values. Example: A Student may have multiple PhoneNumbers.
  • Derived attribute: Computed from other attributes. Example: Age can be derived from DateOfBirth.
  • Simple attribute: Cannot be divided, holds a single value. Example: Gender, BloodGroup

3. Relationship

What it is: Shows association between entities.

Example: Student enrolls in Course.

Types of relationships:

  • One-to-One (1:1): One entity maps to only one entity. Example: A Student has one Passport.
  • One-to-Many (1:N): One entity maps to many entities. Example: A Teacher teaches many Students.
  • Many-to-One (N:1): Many entities map to one entity. Example: Many Students belong to one Department.
  • Many-to-Many (M:N): Many entities map to many entities. Example: Students enroll in many Courses, and Courses have many Students.

4. Mapping ER to Tables (Relations)

  • Strong entity: Directly maps to a table using its primary key.
  • Weak entity: Table with combined key (owner key + partial key) + foreign key reference.
  • One-to-Many: Add a foreign key on the many side.
  • One-to-One: Add a foreign key on either side with UNIQUE constraint.
  • Many-to-Many: Create a junction table with both foreign keys as composite primary key.
  • Multivalued attribute: Map to separate table.
DBMS: Relational Algebra▶

"Relational Algebra is a procedural query language used in databases. It defines operators that transform input tables, or relations, into output tables, and it forms the theoretical foundation of SQL."

  • Selection (σ) filters rows based on a condition. For example, to get students with marks greater than 80, we use selection.
  • Projection (π) chooses specific columns and removes duplicates. For example, to get only the names of students, we project the Name column.
  • Union combines tuples from two compatible tables and removes duplicates. For example, combining two classes’ student lists.
  • Set Difference returns tuples in one table but not in another. For example, students enrolled in Math but not in Physics.
  • Intersection returns tuples present in both tables. For example, students enrolled in both Math and Physics.
  • Cartesian Product pairs each row of one table with every row of another, and it’s the basis for joins. For example, pairing all students with all courses.
  • Rename (ρ) changes table or column names to avoid confusion.
  • Joins combine related tuples from two tables:
    • Theta join uses a condition, like StudentID matching in Student and Enrollment.
    • Equijoin is a theta join with equality condition.
    • Natural join automatically joins tables on columns with the same name.
  • Division is used to find tuples related to all tuples of another relation. For example, finding students who took all courses.
DBMS: Normalization▶

"Normalization is the process of organizing database tables to reduce redundancy and improve data integrity. It means dividing large tables into smaller, related tables and linking them through keys."

1. First Normal Form (1NF)

  • Each column must have atomic (indivisible) values.
  • No repeating groups or arrays.
  • Example: Instead of storing multiple phone numbers in one column, store each number in a separate row.

2. Second Normal Form (2NF)

  • Must be in 1NF.
  • No partial dependency: all non-key columns must depend on the whole primary key.
  • Example: In a table with StudentID + CourseID as a combined key, StudentName should depend only on StudentID, not the combination.

3. Third Normal Form (3NF)

  • Must be in 2NF.
  • No transitive dependency: non-key attributes should not depend on other non-key attributes.
  • Example: If StudentID → Department → DepartmentHead, separate Department and DepartmentHead into a different table.

4. Boyce-Codd Normal Form (BCNF)

  • Stronger than 3NF.
  • Every determinant must be a candidate key, especially when multiple candidate keys exist.
DBMS: Keys▶

"Keys are attributes or combinations of attributes that uniquely identify rows in a table or establish relationships between tables."

  • Primary Key: Uniquely identifies each record in a table and cannot be NULL.
    Example: StudentID in a Student table.
  • Foreign Key: Links one table to another by referencing a primary key.
    Example: StudentID in the Marks table referencing Student table.
  • Candidate Key: All possible attributes that can serve as a primary key.
    Example: Both Email and StudentID could uniquely identify a student, so they are candidate keys.
  • Composite Key: Made by combining two or more columns to uniquely identify a record.
    Example: StudentID + CourseID in an Enrollment table.
  • Alternate Key: A candidate key that was not chosen as the primary key.
    Example: If StudentID is primary, Email can be an alternate key.
  • Super Key: Any key that uniquely identifies a record, even if it has extra attributes.
    Example: StudentID + Email can be a super key because it still uniquely identifies a student.
DBMS: Aggregate Functions▶

"Aggregate functions are used to perform calculations on a set of values and give a single result."

  • COUNT: Counts the number of rows.
    Example: SELECT COUNT(*) FROM students; – gives the total number of students.
  • SUM: Adds up numeric values.
    Example: SELECT SUM(salary) FROM employees; – gives the total salary of all employees.
  • AVG: Returns the average of values.
    Example: SELECT AVG(marks) FROM results; – gives the average marks of students.
  • MAX: Returns the highest value.
    Example: SELECT MAX(price) FROM products; – gives the highest product price.
  • MIN: Returns the lowest value.
    Example: SELECT MIN(age) FROM users; – gives the minimum age among users.
DBMS: Union and Intersection▶

"Union and Intersection are operations used to combine results from multiple queries."

Union

Combines results from two or more SELECT statements and removes duplicates. It returns all rows present in either result set.

Example:

SELECT name FROM students_2024
UNION
SELECT name FROM students_2025;

This gives the names of all students from both batches, without duplicates.

Intersection

Returns only the common records between two queries. Some SQL systems use INTERSECT, or it can be simulated with INNER JOIN.

Example:

SELECT name FROM students_2024
INTERSECT
SELECT name FROM students_2025;

This gives the names of students present in both batches.

DBMS: DBMS vs RDBMS▶

"Both DBMS and RDBMS are used to store and manage data, but there are key differences."

DBMS (Database Management System)

  • Helps store, manage, and retrieve data.
  • Allows basic operations like insert, update, delete, fetch.
  • May not enforce relationships or constraints.
  • Example: Microsoft Access, XML databases, file systems.

RDBMS (Relational DBMS)

  • Stores data in tables with rows and columns.
  • Maintains relationships using keys.
  • Supports SQL and enforces constraints for data integrity.
  • Example: MySQL, Oracle, PostgreSQL, SQL Server.

Key Differences:

  • DBMS may have file-based storage, higher redundancy, and weaker integrity.
  • RDBMS reduces redundancy via normalization and enforces integrity with relationships and constraints.

Redundancy and Data Integrity

  • Redundancy: Repetition of the same data in multiple places. Normalization helps remove this.
  • Data Integrity: Ensures accuracy and consistency of data in the database.

Transitive Dependency

"A transitive dependency occurs when a non-key column depends on another non-key column instead of depending directly on the primary key."

Example: If StudentID → Department and Department → DepartmentHead, then StudentID → DepartmentHead is a transitive dependency.

DBMS: JOINs▶

"Joins are used to combine rows from two or more tables based on a related column between them. They help us fetch meaningful data from multiple tables."

Types of Joins

  • Inner Join:
    Returns only the rows that have matching values in both tables.
    Example: Get students and their enrolled courses, but only if they have enrolled in at least one course.
  • Left (Outer) Join:
    Returns all rows from the left table, and matching rows from the right table. If there’s no match, NULL is returned for the right table.
    Example: List all students and their courses, including students who haven’t enrolled in any course.
  • Right (Outer) Join:
    Returns all rows from the right table, and matching rows from the left table. NULL if no match.
    Example: List all courses and the students enrolled, including courses with no students.
  • Full (Outer) Join:
    Returns all rows from both tables, with NULLs where there’s no match.
    Example: List all students and all courses, showing NULL where no enrollment exists.
  • Cross Join (Cartesian Product):
    Combines every row of one table with every row of the other table.
    Example: Pair every student with every course (used rarely).
  • Self Join:
    A table is joined with itself.
    Example: Find employees and their managers from the same Employee table.
OOPS: Core Concept▶

OOPS, or Object-Oriented Programming, is a programming paradigm that organizes code around objects rather than just functions and logic. Each object represents real-world entities and contains data and behavior.

OOPS helps make code:

  • Modular: so different parts are separated;
  • Reusable: so we can use objects or classes in multiple places;
  • Easy to understand: because it models real-world entities;
  • Easy to maintain: since changes in one object usually don’t affect others.
OOPS: Abstraction▶

Definition: "Abstraction is the concept of hiding the internal details of an object and exposing only the necessary features to the outside world. It allows programmers to focus on what an object does, rather than how it does it."

Key Points:

  • Abstraction simplifies complex systems by hiding unnecessary details.
  • It helps reduce code complexity and makes programs easier to understand and maintain.
  • Abstraction is achieved in OOPS using:
    • Abstract Classes – Classes that cannot be instantiated directly and may contain abstract methods.
    • Interfaces (in languages like Java) – Define methods without implementation, which must be implemented by child classes.

Real-world Example:

  • Driving a car: You press the accelerator to increase speed. You don’t need to know how the engine works internally.
  • The remote control of a TV: You press buttons to change channels or volume. You don’t need to know the circuits inside.

Code Example in Python:

from abc import ABC, abstractmethod

# Abstract class
class Vehicle(ABC):
    @abstractmethod
    def start(self):
        pass
        
    @abstractmethod
    def stop(self):
        pass

# Child class implements the abstract methods
class Car(Vehicle):
    def start(self):
        print("Car is starting...")
        
    def stop(self):
        print("Car has stopped.")

# Using the class
my_car = Car()
my_car.start()  # Output: Car is starting...
my_car.stop()   # Output: Car has stopped.

Benefits of Abstraction:

  • Reduces complexity – Users don’t see unnecessary details.
  • Improves maintainability – Changes in internal implementation don’t affect users.
  • Enhances security – Sensitive internal details are hidden.
OOPS: Encapsulation▶

Definition: "Encapsulation is the concept of wrapping data (variables) and methods that operate on the data into a single unit, usually a class. It also restricts direct access to the data by using private or protected members, so the internal state of an object is protected."

Key Points:

  • Encapsulation hides the internal details of an object.
  • Access to data is controlled using getter and setter methods.
  • It improves security, modularity, and maintainability.

Real-world Example:

A capsule medicine contains medicine inside a protective shell. You only use it as intended, without knowing or touching what’s inside. Similarly, in a program, encapsulation hides internal variables and exposes only safe methods to access or modify them.

Python Code Example:

class BankAccount:
    def __init__(self, balance):
        self.__balance = balance  # private variable
    
    # Getter method
    def get_balance(self):
        return self.__balance
        
    # Setter method
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
        else:
            print("Invalid amount!")

# Using the class
account = BankAccount(1000)
print(account.get_balance())  # Output: 1000
account.deposit(500)
print(account.get_balance())  # Output: 1500
# Direct access is not allowed: account.__balance -> Error

Benefits of Encapsulation:

  • Protects data from unauthorized access.
  • Improves code modularity – the class can be maintained independently.
  • Easier debugging and maintenance – all operations on data are controlled.
OOPS: Inheritance▶

Definition: "Inheritance is a concept in OOPS that allows one class, called the child class, to inherit properties and methods from another class, called the parent class. This helps in reusing code and reducing redundancy."

Key Points:

  • Promotes code reusability – common code is written once in the parent class.
  • Helps in extending functionality – child class can add new features.
  • Supports hierarchical relationships between classes.

Real-world Example:

A Vehicle class can have properties like speed and fuel. A Car class can inherit these properties and methods, and add its own features like number_of_doors.

Python Code Example:

# Parent class
class Vehicle:
    def __init__(self, brand, speed):
        self.brand = brand
        self.speed = speed
        
    def show_speed(self):
        print(f"{self.brand} is running at {self.speed} km/h")

# Child class
class Car(Vehicle):
    def __init__(self, brand, speed, doors):
        super().__init__(brand, speed)  # inherit properties from Vehicle
        self.doors = doors
        
    def show_doors(self):
        print(f"{self.brand} has {self.doors} doors")

# Using the classes
my_car = Car("Toyota", 120, 4)
my_car.show_speed()   # Output: Toyota is running at 120 km/h
my_car.show_doors()   # Output: Toyota has 4 doors
OOPS: Polymorphism▶

Definition: "Polymorphism means 'many forms'. It allows the same function or method name to behave differently depending on the object that calls it. This makes code more flexible and easier to maintain."

Key Points:

  • Achieved through method overriding (Python) and method overloading (other languages).
  • Lets objects of different classes respond differently to the same method call.
  • Helps in code reusability and extensibility.

1. Method Overriding (Runtime Polymorphism)

Happens when a child class defines a method that already exists in its parent class but provides a new implementation. The method name and parameters must be the same. Decided at runtime.

Python Example:

class Animal:
    def speak(self):
        print("Animal speaks")

class Dog(Animal):
    def speak(self):
        print("Dog barks")

a = Animal()
d = Dog()
a.speak()  # Output: Animal speaks
d.speak()  # Output: Dog barks

2. Method Overloading

Means having multiple methods with the same name but different parameters. Python does not support traditional compile-time overloading; the last defined method overrides previous ones.

Python Example:

class MathOps:
    def add(self, a, b):
        print(a + b)
        
    def add(self, a, b, c):
        print(a + b + c)

obj = MathOps()
obj.add(2, 3, 4)  # Output: 9
# obj.add(2, 3) # This would cause an error

Explanation: The first add(a, b) method is overridden by the second add(a, b, c) method. Python only keeps the last defined method.

OOPS: Class and Object▶

Definition: "In OOPS, a class is like a blueprint or template for creating objects. It defines the attributes (data) and behaviors (methods) that its objects will have. An object is a real-world instance of that class, created using the blueprint."

Python Example:

class Student:
    def __init__(self, name, age):
        self.name = name  # attribute
        self.age = age    # attribute
        
    def show(self):       # method (behavior)
        print("Name:", self.name, "Age:", self.age)

# Creating an object
stu1 = Student("Jin", 21)
stu1.show()  # Output: Name: Jin Age: 21

Explanation: Student is the class (blueprint). stu1 is an object (instance). show() is a method.

Real-world Analogy:

  • Class: A blueprint of a house.
  • Object: A house built using that blueprint.
OOPS: Constructor▶

Definition: "A constructor is a special method in a class that is automatically called when an object is created. It is mainly used to initialize object variables with default or user-provided values."

Key Points:

  • Automatically invoked when an object is created.
  • Used to initialize attributes of the object.
  • In Python, the constructor is named __init__.

Python Example:

class Student:
    def __init__(self, name, age):  # Constructor
        self.name = name
        self.age = age
        
    def show(self):
        print("Name:", self.name, "Age:", self.age)

stu1 = Student("Jin", 21)  # __init__ called automatically
stu1.show()                 # Output: Name: Jin Age: 21

Types of Constructors

Default Constructor:

Takes no parameters except self. Python provides it automatically if no constructor is defined.

class Student:
    def __init__(self):
        self.name = "Unknown"
        self.age = 0
stu = Student()
print(stu.name, stu.age)  # Output: Unknown 0

Parameterized Constructor:

Takes parameters to initialize object attributes.

stu1 = Student("Jin", 21)
print(stu1.name, stu1.age)  # Output: Jin 21

Constructor Overloading (Simulation in Python):

Python doesn’t support traditional overloading, but we simulate it using default arguments.

class Student:
    def __init__(self, name="Unknown", age=0):
        self.name = name
        self.age = age
        
stu1 = Student()
stu2 = Student("Jin", 21)
print(stu1.name, stu1.age)  # Output: Unknown 0
print(stu2.name, stu2.age)  # Output: Jin 21
