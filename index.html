OOP stands for Object-Oriented Programming. It’s a way of writing programs that focuses on objects rather than just functions or logic. Each object represents a real-world entity and has its own data and behavior. In OOP, we group related variables and methods into one unit called a class, and we create objects from that class to perform tasks. Example: 
class Car {
   void start() {
      System.out.println("Car started");
   }
}
public class Main {
   public static void main(String[] args) {
       Car myCar = new Car(); // object creation
       myCar.start();
   }
}
Here, Car is a class, and myCar is an object of that class.
The four main principles of OOP are: 1. Encapsulation: It means wrapping data (variables) and code (methods) into a single unit, i.e., a class. It also hides the data using private access modifiers and allows access through getter and setter methods. Example: 
class Student {
   private int age;
   public void setAge(int a) { age = a; }
   public int getAge() { return age; }
}
Here, the age variable is hidden from outside the class — that’s encapsulation. --- 2. Inheritance: It allows one class to use the properties and methods of another class using the extends keyword. It promotes code reusability. Example: 
class Animal {
   void eat() { System.out.println("Eating..."); }
}
class Dog extends Animal {
   void bark() { System.out.println("Barking..."); }
}
Here, the Dog class inherits the eat() method from Animal. --- 3. Polymorphism: Polymorphism means “many forms.” It allows the same method to perform different actions depending on the object that calls it. It can be achieved through method overloading and method overriding. Example: Overloading (compile-time): 
void show(int a) {}
void show(String b) {}
Overriding (runtime): 
class A { void display() { System.out.println("A"); } }
class B extends A { void display() { System.out.println("B"); } }
--- 4. Abstraction: Abstraction hides unnecessary details and shows only essential information. It can be achieved using abstract classes or interfaces. Example: 
abstract class Shape {
   abstract void draw();
}
class Circle extends Shape {
   void draw() { System.out.println("Drawing Circle"); }
}
Here, we only show what to do (draw), not how — that’s abstraction.

"Normalization is the process of organizing database tables to reduce redundancy and improve data integrity. It means dividing large tables into smaller, related tables and linking them through keys." 1. First Normal Form (1NF) • Each column must have atomic (indivisible) values. • No repeating groups or arrays. • Example: Instead of storing multiple phone numbers in one column, store each number in a separate row. 2. Second Normal Form (2NF) • Must be in 1NF. • No partial dependency: all non-key columns must depend on the whole primary key. • Example: In a table with StudentID + CourseID as a combined key, StudentName should depend only on StudentID, not the combination. 3. Third Normal Form (3NF) • Must be in 2NF. • No transitive dependency: non-key attributes should not depend on other non-key attributes. • Example: If StudentID → Department → DepartmentHead, separate Department and DepartmentHead into a different table. 4. Boyce-Codd Normal Form (BCNF) • Stronger than 3NF. • Every determinant must be a candidate key, especially when multiple candidate keys exist.
