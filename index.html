In C, the main() function is the starting point of any program. Every C program must have a main() function, otherwise the program won’t run.

It is the function where execution begins. It can return an integer value, usually 0, which indicates that the program executed successfully.

#include <stdio.h>
int main() {
    printf("Hello World\n");
    return 0;
}

Here, the program starts from main(), prints Hello World, and returns 0. Sometimes main() can also accept arguments like int argc and char *argv[]. These are used to take command-line inputs.

Key points to mention in an interview:

  • main() is mandatory.
  • Execution always starts here.
  • Can return int or sometimes void (though int is standard).
  • Can accept command-line arguments.

A pointer in C is a variable that stores the memory address of another variable.

  • Normal variable: stores value
  • Pointer variable: stores address (location of the value)

For example, if I have int a = 10;, I can declare a pointer as int *ptr = &a;. Here, ptr stores the address of a.

I can access the value of a using the pointer by dereferencing it: *ptr gives 10.

Pointers are powerful because they let us:

  • Directly manipulate memory.
  • Pass large data structures to functions efficiently.
  • Handle arrays dynamically.

We can also do pointer arithmetic. For example, if ptr points to an array element, ptr + 1 points to the next element.

int arr[3] = {10, 20, 30};
int *ptr = arr;
printf("%d", *(ptr + 1)); // Prints 20

Pointers are essential in C because they provide flexibility and efficiency.

Normally, memory is allocated at compile time (static allocation). But sometimes we don't know how much memory we'll need. Dynamic Memory Allocation (DMA) lets us allocate memory at runtime using functions from the <stdlib.h> library.

1. malloc() – Memory Allocation

malloc() stands for memory allocation. It allocates a single block of memory of a given size in bytes. The memory is not initialized, so it may contain garbage values.

ptr = (castType*) malloc(sizeInBytes);

// Example:
int *ptr;
ptr = (int*) malloc(5 * sizeof(int)); // Allocate memory for 5 integers

if(ptr == NULL) {
    printf("Memory allocation failed");
}

2. calloc() – Contiguous Allocation

calloc() stands for contiguous allocation. It allocates memory for multiple elements and initializes all bits to zero.

ptr = (castType*) calloc(numElements, sizeOfElement);

// Example:
int *ptr;
ptr = (int*) calloc(5, sizeof(int)); // 5 integers, all initialized to 0

3. realloc() – Reallocation

realloc() is used to resize previously allocated memory. This is useful if we initially allocated memory but later need more or less.

ptr = (castType*) realloc(ptr, newSizeInBytes);

// Example:
ptr = (int*) realloc(ptr, 10 * sizeof(int)); // Resize memory to hold 10 integers

4. free() – Deallocate Memory

free() is used to release memory that was previously allocated by malloc(), calloc(), or realloc().

free(ptr);
ptr = NULL; // Avoid dangling pointer

It is essential to free memory to prevent memory leaks.

In C, a static function is a function whose scope is limited to the file in which it is declared. That means it cannot be accessed from other files, even if those files are part of the same project.

The main use of static functions is encapsulation. It helps hide internal logic and prevents other files from using or interfering with it. This also avoids name conflicts if multiple files have functions with the same name.

A static function retains its lifetime throughout the program execution, just like a global variable. But unlike global functions, it is not visible outside its file.

#include <stdio.h>

static void greet() {
    printf("Hello from static function!\n");
}

int main() {
    greet(); // Works fine
    return 0;
}

If I try to call greet() from another file, it will not work because it is static. So, to summarize, a static function is file-private, cannot be accessed externally, and is useful for hiding internal helper functions inside a file.

Local Variable

A local variable is a variable that is declared inside a function or a block. It only exists while the function is running. When the function ends, the variable is destroyed and its memory is freed.

void func() {
    int x = 10; // Local variable
    printf("%d", x);
}

Here, x exists only inside func() and cannot be accessed outside it. Local variables are useful because they keep data private to the function and prevent unwanted changes from other parts of the program.

Global Variable

A global variable is declared outside all functions, usually at the top of the program. It can be accessed by any function in the same file. If we use the extern keyword, it can even be accessed from other files.

#include <stdio.h>

int count = 0; // Global variable

void increment() {
    count++;
}

int main() {
    increment();
    printf("%d", count); // Prints 1
    return 0;
}

Global variables are useful for sharing data between functions, but they should be used carefully because any function can change them, which may cause bugs.

In C, control statements help us control the flow of the program. There are four main ones: break, continue, goto, and return.

1. break

  • break is used to exit from a loop or switch immediately.
  • Example: 
    for(int i = 1; i <= 5; i++) {
    if(i == 3) break;
    printf("%d ", i);
}
// Output: 1 2
  • Here, the loop stops when i becomes 3.

2. continue

  • continue skips the current iteration and goes to the next iteration of the loop.
  • Example: 
    for(int i = 1; i <= 5; i++) {
    if(i == 3) continue;
    printf("%d ", i);
}
// Output: 1 2 4 5
  • Here, 3 is skipped, but the loop continues.

3. goto

  • goto jumps to a labeled part of the program.
  • Example: 
    int i = 1;
label:
    printf("%d ", i);
    i++;
    if(i <= 3) goto label;
// Output: 1 2 3
  • goto can make code messy, so it’s usually avoided in structured programming.

4. return

  • return exits from a function and optionally returns a value.
  • Example: 
    int sum(int a, int b) {
    return a + b;
}
int main() {
    printf("%d", sum(2,3)); // Output: 5
    return 0;
}
  • It is used to send values back to the caller and end function execution.

In C, a predefined function (or library function) is a function that is already provided by C in its standard library. We don’t need to write the code for it; we just call it to perform a specific task. These functions are declared in header files like stdio.h, math.h, string.h, etc.

Categories:

  1. Input/Output Functions (stdio.h)
    • Examples: printf(), scanf().
  2. Mathematical Functions (math.h)
    • Examples: sqrt(), pow(), sin(), cos().
  3. String Handling Functions (string.h)
    • Examples: strlen(), strcpy(), strcmp().
  4. Character Handling Functions (ctype.h)
    • Examples: isdigit(), isalpha(), toupper().
  5. Time and Date Functions (time.h)
    • Examples: time(), clock(), difftime().
  6. Other Utility Functions (stdlib.h)
    • Examples: malloc(), calloc(), exit().

In C, header files contain predefined library functions. We can use these functions in our program by including the header file using #include. For example, #include <stdio.h> lets us use printf() and scanf().

Header files help reduce the amount of code we need to write and make the program simpler and less complex. So basically, whenever we want to use standard functions like input/output, math calculations, string operations, or memory management, we just include the relevant header file instead of writing the function ourselves.

In C, an array is a collection of elements of the same data type, stored in contiguous memory locations. We use arrays when we want to store multiple values of the same type together.

data_type array_name[size];
// Example:

In C, the const keyword is used to declare a constant variable, which cannot be changed after it is initialized.

const int x = 10;

Here, x is a constant, so if we try to assign a new value to x, the compiler will give an error. Using const helps prevent accidental changes to variables that should remain constant, making the program safer and more reliable.

In C, a memory leak occurs when a program allocates memory dynamically but fails to free it after use. The memory becomes unreachable, but it still occupies space in RAM. Over time, this can cause the program or even the system to slow down or crash.

Cause: Repeatedly allocating memory using functions like malloc() or calloc() without freeing the old memory.

Solution: Always use free() to release dynamically allocated memory once it is no longer needed.

int *ptr = (int*) malloc(sizeof(int));
// use the memory
free(ptr); // frees the memory to prevent memory leak

In C, there are two ways to pass arguments to a function: call by value and call by reference.

1. Call by Value

  • The function receives a copy of the variable.
  • Any changes inside the function do not affect the original variable.
  • Safer because the original data cannot be changed, but it uses extra memory for the copy.
  • Example: 
    void fun(int x) {
    x = x + 10;
}
int main() {
    int a = 5;
    fun(a);
    printf("%d", a); // Prints 5
}
  • Here, a remains 5 outside the function because only a copy was modified.

2. Call by Reference

  • The function receives the address of the variable.
  • Any changes inside the function affect the original variable.
  • Example: 
    void fun(int *x) {
    *x = *x + 10;
}
int main() {
    int a = 5;
    fun(&a);
    printf("%d", a); // Prints 15
}
  • Here, a becomes 15 because the function modified the original variable through its address.

1. Compiler

  • The compiler compiles the entire code at once.
  • It displays all errors together after compilation.
  • Programs compiled by a compiler generally have faster execution.
  • Example: gcc for C.

2. Interpreter

  • The interpreter executes the program line by line.
  • It shows output or errors for each line during execution.
  • Programs executed by an interpreter generally have slower execution.
  • Example: Python interpreter, JavaScript engine.

In C, loops are used to execute a block of statements repeatedly. There are two types of loops based on when the condition is checked:

1. Entry Loop (e.g., while loop)

  • In an entry loop, the condition is checked first.
  • The statements inside the loop execute only if the condition is true.
  • int i = 1;
while(i <= 3) {
    printf("%d ", i);
    i++;
}
// Output: 1 2 3

2. Exit Loop (e.g., do-while loop)

  • In an exit loop, the statements are executed first, and then the condition is checked at the end.
  • It is also called an exit-controlled loop.
  • The loop executes at least once, even if the condition is false.
  • int i = 1;
do {
    printf("%d ", i);
    i++;
} while(i <= 3);
// Output: 1 2 3

In C, typecasting is the process of converting a variable from one data type to another. We use it when we want to temporarily change the type of a variable, usually for calculations or compatibility.

1. Implicit Typecasting (Type Conversion)

  • Done automatically by the compiler.
  • Usually occurs when a smaller type is converted to a larger type.
  • Example: 
    int a = 10;
float b = a; // int automatically converted to float

2. Explicit Typecasting (Type Conversion)

  • Done manually by the programmer using parentheses.
  • Example: 
    float a = 9.8;
int b = (int) a; // float explicitly converted to int (b becomes 9)

In C, the sizeof operator is used to determine the exact size of a data type or variable in bytes. It can be applied to built-in data types like int, char, float, or even arrays and structures.

int a;
printf("%zu", sizeof(a));  // Prints the size of int, usually 4 bytes
printf("%zu", sizeof(char)); // Prints 1 byte

Using sizeof helps understand memory usage and is useful in dynamic memory allocation. (%zu is the format specifier for size_t, which is the type returned by sizeof).

In C, a structure is a user-defined data type that allows us to group variables of different data types together under a single name. Each member of a structure has its own memory, and we can access them individually using the dot operator (.).

struct Student {
    int id;
    char name[30];
    float marks;
};

int main() {
    struct Student s1;
    s1.id = 101;        // Accessing integer member
    strcpy(s1.name, "Viji"); // Accessing string member
    s1.marks = 95.5;      // Accessing float member
    return 0;
}

Structures are useful when we want to store related information together, like a student's ID, name, and marks.

In C, a macro is a preprocessor directive defined using #define. Before compilation, the macro name is replaced by its value wherever it appears in the code. Using macros can reduce code length and make programs easier to maintain.

Here, PI and SQUARE(x) are replaced by their values before compilation.

Normalization is the process of organizing database tables to reduce redundancy and improve data integrity. It means dividing large tables into smaller, related tables and linking them through keys.

  1. First Normal Form (1NF)
    • Each column must have atomic (indivisible) values.
    • No repeating groups or arrays.
    • Example: Instead of storing multiple phone numbers in one column, store each number in a separate row.
  2. Second Normal Form (2NF)
    • Must be in 1NF.
    • No partial dependency: all non-key columns must depend on the whole primary key (only relevant for composite keys).
    • Example: In a table with (StudentID + CourseID) as key, StudentName should depend only on StudentID, not the combination. So, StudentName should be in a separate Students table.
  3. Third Normal Form (3NF)
    • Must be in 2NF.
    • No transitive dependency: non-key attributes should not depend on other non-key attributes.
    • Example: If StudentID → Department → DepartmentHead, this is a transitive dependency. DepartmentHead should be in a Department table, not the Student table.
  4. Boyce-Codd Normal Form (BCNF)
    • Stronger than 3NF.
    • Every determinant must be a candidate key, especially when multiple candidate keys exist.

Joins are used to combine rows from two or more tables based on a related column between them. They help us fetch meaningful data from multiple tables.

Types of Joins

  1. Inner Join: Returns only the rows that have matching values in both tables.
  2. Left (Outer) Join: Returns all rows from the left table, and matching rows from the right table. If there’s no match, NULL is returned for the right table.
  3. Right (Outer) Join: Returns all rows from the right table, and matching rows from the left table. NULL if no match.
  4. Full (Outer) Join: Returns all rows from both tables, with NULLs where there’s no match.
  5. Cross Join (Cartesian Product): Combines every row of one table with every row of the other table.
  6. Self Join: A table is joined with itself. (e.g., to find employees and their managers from the same Employee table).

Transactions follow ACID properties to ensure reliability:

  1. Atomicity – All or nothing. A transaction must complete fully or not at all.
    • Example: If debiting A succeeds but crediting B fails, the debit is undone (rolled back).
  2. Consistency – Data must remain valid. Database rules and constraints are preserved.
    • Example: Total balance across accounts remains the same before and after transfer.
  3. Isolation – Transactions do not interfere with each other. Each runs as if it is the only one.
    • Example: Two users cannot book the same seat at the same time.
  4. Durability – Once committed, changes are permanent. They survive system failures.
    • Example: After a successful transfer, the records are preserved even if the system restarts.

Definition: An ER model is a way to show how data is structured in a database. It uses entities, attributes, and relationships to give a high-level conceptual view of the data.

  1. Entity: A real-world object. (e.g., Student, Car).
    • Strong entity: Has its own primary key (e.g., Student with StudentID).
    • Weak entity: Depends on a strong entity (e.g., Dependent of an Employee).
  2. Attribute: A property that describes an entity.
    • Key attribute: Uniquely identifies an entity (e.g., StudentID).
    • Composite attribute: Can be divided (e.g., Name → FirstName + LastName).
    • Multivalued attribute: Can have multiple values (e.g., PhoneNumbers).
    • Derived attribute: Computed from other attributes (e.g., Age from DateOfBirth).
  3. Relationship: Shows association between entities (e.g., Student enrolls in Course).
    • One-to-One (1:1): A Student has one Passport.
    • One-to-Many (1:N): A Teacher teaches many Students.
    • Many-to-Many (M:N): Students enroll in many Courses.

In DBMS, a transaction is a sequence of operations performed as a single logical unit of work. The key idea is: either all operations succeed or none are applied, which maintains database consistency.

Example of a Transaction

BEGIN;
UPDATE Account SET Balance = Balance - 1000 WHERE AccNo = 'A';
UPDATE Account SET Balance = Balance + 1000 WHERE AccNo = 'B';
COMMIT;
  • If any step fails, ROLLBACK undoes all changes.
  • Transaction control commands include: COMMIT (save changes), ROLLBACK (undo changes), and SAVEPOINT (set a point to roll back to).

In DBMS, a Primary Key is a column or a set of columns in a table that uniquely identifies each record. A primary key cannot have NULL values, and no two rows can have the same primary key.

CREATE TABLE Student (
    RollNo INT PRIMARY KEY,
    Name VARCHAR(50),
    Age INT
);

Here, RollNo is the primary key, so each student has a unique roll number.

In a database, constraints are rules applied on columns to ensure data integrity.

1. NULL Constraint

  • Determines whether a column can accept NULL values.
  • Use NOT NULL to make a column mandatory.
  • Example: Name VARCHAR(50) NOT NULL (The Name column cannot be left empty).

2. UNIQUE Constraint

  • Ensures that all values in a column are distinct (no duplicates).
  • Most DBMS allow one NULL value in a UNIQUE column (as NULL is not equal to anything, including other NULLs).
  • Example: Email VARCHAR(50) UNIQUE (No two rows can have the same email address).

In DBMS, the degree of a relation is the number of attributes or columns in a table. It tells us how many fields are used to describe each record in the table.

CREATE TABLE Student (
    RollNo INT,
    Name VARCHAR(50),
    Age INT,
    Marks INT
);

Here, the Student table has 4 columns: RollNo, Name, Age, Marks. So, the degree of this relation is 4. Degree depends on the number of columns, not the number of rows.

Both WHERE and HAVING are used to filter records, but they work at different stages:

1. WHERE Clause

  • Filters individual rows BEFORE grouping.
  • Cannot use aggregate functions (like SUM, AVG, COUNT).
  • Appears before GROUP BY.
  • Example: 
    SELECT * FROM Sales
WHERE Amount > 1000;

2. HAVING Clause

  • Filters groups of rows AFTER grouping.
  • Can use aggregate functions.
  • Appears after GROUP BY.
  • Example: 
    SELECT Seller, SUM(Amount)
FROM Sales
GROUP BY Seller
HAVING SUM(Amount) > 5000;

Relational Algebra is a procedural query language used in databases. It defines operators that transform input tables (relations) into output tables, and it forms the theoretical foundation of SQL.

  • Selection (σ): Filters rows based on a condition.
  • Projection (π): Chooses specific columns and removes duplicates.
  • Union: Combines tuples from two compatible tables (removes duplicates).
  • Set Difference: Returns tuples in one table but not in another.
  • Intersection: Returns tuples present in both tables.
  • Cartesian Product: Pairs each row of one table with every row of another (basis for joins).
  • Rename (ρ): Changes table or column names.
  • Joins: Combine related tuples (Theta join, Equijoin, Natural join).

Keys are attributes that uniquely identify rows in a table or establish relationships.

  • Primary Key: Uniquely identifies each record in a table (cannot be NULL).
    • Example: StudentID in a Student table.
  • Foreign Key: Links one table to another by referencing a primary key.
    • Example: StudentID in the Marks table referencing Student table.
  • Candidate Key: All possible attributes that can serve as a primary key.
    • Example: Both Email and StudentID could be candidate keys.
  • Composite Key: A key made by combining two or more columns.
    • Example: (StudentID + CourseID) in an Enrollment table.
  • Alternate Key: A candidate key that was not chosen as the primary key.
    • Example: If StudentID is primary, Email can be an alternate key.
  • Super Key: Any key that uniquely identifies a record (even if it has extra attributes).
    • Example: (StudentID + Email) is a super key.

Aggregate functions perform calculations on a set of values and give a single result.

  • COUNT: Counts the number of rows.

  • SUM: Adds up numeric values.
    • SELECT SUM(salary) FROM employees;
  • AVG: Returns the average of values.
    • SELECT AVG(marks) FROM results;

    • SELECT MAX(price) FROM products;
  • MIN: Returns the lowest value.
    • SELECT MIN(age) FROM users;

  • Union: Combines results from two or more SELECT statements and removes duplicates. It returns all rows present in either result set.

UNION
SELECT name FROM students_2025;
  • Intersection: Returns only the common records between two queries.

  • DBMS (Database Management System): Helps store, manage, and retrieve data. May not enforce relationships or constraints.
    • Example: XML databases, file systems.
  • RDBMS (Relational DBMS): Stores data in tables with rows and columns. Maintains relationships using keys. Supports SQL and enforces constraints.

Key Difference: RDBMS is based on the relational model (tables/keys), reduces redundancy via normalization, and enforces integrity. DBMS is a more general term.

  • Redundancy: Repetition of the same data in multiple places. Normalization helps remove this.
  • Data Integrity: Ensures accuracy and consistency of data (e.g., a foreign key prevents an order from referencing a non-existing customer).
  • Transitive Dependency: Occurs when a non-key column depends on another non-key column (e.g., StudentID → Department → DepartmentHead). This is removed in 3NF.

OOPS, or Object-Oriented Programming, is a programming paradigm that organizes code around objects rather than just functions and logic. Each object represents real-world entities and contains data (attributes) and behavior (methods).

OOPS helps make code:

  • Modular: Different parts are separated.
  • Reusable: We can use objects or classes in multiple places.
  • Easy to understand: It models real-world entities.
  • Easy to maintain: Changes in one object usually don’t affect others.

Abstraction is the concept of hiding the internal details of an object and exposing only the necessary features. It allows programmers to focus on what an object does, rather than how it does it.

Real-world Example: When you drive a car, you press the accelerator to increase speed. You don’t need to know how the engine works internally.

In OOPS, abstraction is achieved using Abstract Classes and Interfaces.

# Python Example
from abc import ABC, abstractmethod

class Vehicle(ABC):  # Abstract class
    @abstractmethod
    def start(self):
        pass

class Car(Vehicle):
    def start(self):
        print("Car is starting...")

my_car = Car()
my_car.start()  # User only interacts with start()

Encapsulation is the concept of wrapping data (variables) and methods that operate on the data into a single unit (a class). It also restricts direct access to the data by using private members. Access is controlled using getter and setter methods.

Real-world Example: A capsule medicine contains medicine inside a protective shell. You only use it as intended, without touching what’s inside.

# Python Example
class BankAccount:
    def __init__(self, balance):
        self.__balance = balance  # private variable

    # Getter method
    def get_balance(self):
        return self.__balance

    # Setter method
    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount

account = BankAccount(1000)
print(account.get_balance()) # Output: 1000
# print(account.__balance) # This would cause an error

Inheritance allows one class (the child class) to inherit properties and methods from another class (the parent class). This helps in code reusability.

Real-world Example: A Car class can inherit properties like speed and fuel from a Vehicle class, and add its own features like number_of_doors.

# Python Example
class Vehicle:  # Parent class
    def __init__(self, brand):
        self.brand = brand
    
    def show_speed(self):
        print(f"{self.brand} is running")

class Car(Vehicle):  # Child class
    def __init__(self, brand, doors):
        super().__init__(brand)  # inherit from Vehicle
        self.doors = doors

my_car = Car("Toyota", 4)
my_car.show_speed()  # Output: Toyota is running

Polymorphism means "many forms". It allows the same function or method name to behave differently depending on the object that calls it. This makes code more flexible.

This is achieved through Method Overriding (Runtime Polymorphism).

Method Overriding: Happens when a child class defines a method that already exists in its parent class, providing a new implementation.

# Python Example
class Animal:
    def speak(self):
        print("Animal speaks")

class Dog(Animal):
    def speak(self):  # Overriding the parent method
        print("Dog barks")

a = Animal()
d = Dog()

a.speak()  # Output: Animal speaks
d.speak()  # Output: Dog barks

Method Overloading (defining multiple methods with the same name but different parameters) is not directly supported in Python as it is in Java. Python uses default arguments or *args to achieve similar functionality.

Java has many strong features:

  • Simple: Easy to learn, syntax is similar to C++ but removed confusing features like pointers.
  • Object-Oriented: Everything in Java is based on objects and classes.
  • Platform Independent: Java code is compiled into bytecode, which can run on any machine that has a JVM. This is called “Write Once, Run Anywhere.”
  • Secure: Java doesn’t use pointers and runs inside a virtual machine (sandbox).
  • Robust: It has strong memory management, exception handling, and automatic garbage collection.
  • Multithreaded: Java supports multiple threads, meaning it can perform several tasks at the same time.
  • JDK (Java Development Kit): Used to develop Java applications. It includes the compiler, debugger, and other tools. It also contains the JRE.
  • JRE (Java Runtime Environment): Provides the environment needed to run Java programs. It includes the JVM and the core libraries.
  • JVM (Java Virtual Machine): The heart of Java. It actually runs the Java bytecode and converts it into machine code. It also manages memory and garbage collection.

In simple terms: To write and compile code, I use the JDK. To just run the code, I need the JRE. The JVM is what executes the code inside the JRE.

Platform independence means Java programs can run on any operating system without changing the code. This happens because when we compile a Java program, it doesn’t turn into machine-specific code. Instead, it becomes bytecode (a .class file). This bytecode can run on any system (Windows, Mac, Linux) that has a JVM installed. That’s why Java is called “Write Once, Run Anywhere.”

In Java, the main() method is the entry point of every standalone program. The program starts its execution from here.

public static void main(String[] args)

Here’s what each word means:

  • public: Accessible by the JVM from anywhere.
  • static: The method can be called without creating an object of the class.
  • void: The method doesn’t return any value.
  • main: The method name recognized by the JVM.
  • String[] args: Used to accept command-line inputs as an array of strings.

Java has two categories of data types:

1. Primitive Data Types

These are the basic types built into the language. They store the actual value directly in memory.

  • int, long, short, byte (for whole numbers)
  • float, double (for decimal numbers)
  • char (for single characters)
  • boolean (for true or false)

2. Non-Primitive Data Types (Reference Types)

These store references (memory addresses) to the data, not the actual value. They are created by the programmer (except for String).

  • String
  • Arrays
  • Classes and Interfaces
  1. Local Variable: Declared inside a method or block. It must be initialized before use and is only accessible within that method.
  2. Instance Variable: Declared inside a class but outside any method. Each object of the class gets its own copy.
  3. Static Variable: Declared with the static keyword. It is shared by all objects of the class. There is only one copy of this variable.
class Student {
    int rollNo; // Instance variable
    static String college = "ABC College"; // Static variable
    
    void show() {
        int a = 10; // Local variable
    }
}

In Java, the == operator and the .equals() method are used for comparison, but they do different things:

  • == operator: Checks if two references point to the same memory location. It compares the addresses.
  • .equals() method: Checks if two objects have the same content (value).
String s1 = new String("Java");
String s2 = new String("Java");

System.out.println(s1 == s2);       // false - they are different objects in memory
System.out.println(s1.equals(s2));  // true - their content is the same

Type casting means converting one data type into another.

  1. Widening (Automatic Casting): Happens automatically when converting a smaller type to a larger type (e.g., int to double). No data loss. 
    int a = 10;
double b = a; // automatically converts
System.out.println(b); // 10.0
  2. Narrowing (Explicit Casting): Requires manual conversion using parentheses when converting a larger type to a smaller type (e.g., double to int). May cause data loss. 
    double x = 9.78;
int y = (int) x; // manually converts
System.out.println(y); // 9 (data is lost)

Method Overloading (Compile-time Polymorphism)

  • Having multiple methods in the same class with the same name but different parameters (different type, number, or order of parameters).
  • Happens at compile time.
  • class Display {
    void show(int a) { System.out.println(a); }
    void show(String b) { System.out.println(b); }
}

Method Overriding (Runtime Polymorphism)

  • Redefining a method of the parent class in the child class.
  • The method name, return type, and parameters must be the same.
  • Happens at runtime.
  • class Animal {
    void sound() { System.out.println("Animal sound"); }
}
class Dog extends Animal {
    @Override
    void sound() { System.out.println("Dog barks"); }
}

A constructor is a special method used to initialize objects. It has the same name as the class and no return type. It is called automatically when an object is created using the new keyword.

Types of Constructors:

  1. Default Constructor: Created automatically by the compiler if we don’t define one. It takes no arguments.
  2. Parameterized Constructor: Accepts arguments to initialize instance variables. 
    class Student {
    String name;
    Student(String n) { // Parameterized constructor
        name = n;
    }
}
Student s1 = new Student("Viji");

Abstract Class

  • Used to achieve partial abstraction (0 to 100%).
  • Can have both abstract (no body) and non-abstract (concrete) methods.
  • Can have constructors, instance variables, and static methods.
  • A class can extend only one abstract class.
  • Use an abstract class when you want to share common code among subclasses.

Interface

  • Used to achieve full abstraction (100%).
  • Can only have abstract methods (before Java 8). Java 8+ allows default and static methods.
  • Variables are public static final by default.
  • A class can implement multiple interfaces.
  • Use interfaces when you want to define a contract/behavior that multiple unrelated classes can follow.

this keyword

this is a reference variable that refers to the current object of the class. It is used to:

  1. Differentiate between instance variables and local parameters (when they have the same name). 
    class Student {
    int id;
    Student(int id) {
        this.id = id; // this.id is instance var, id is parameter
    }
}
  2. Call the current class's constructor (this()).

super keyword

super is a reference variable that refers to the parent class object. It is used to:

  1. Call the parent class constructor (super()). This must be the first line in the child constructor. 
    class Dog extends Animal {
    Dog() {
        super(); // calls parent class constructor
    }
}
  2. Access parent class methods or variables (super.methodName()).
  • final (keyword): Used to make variables constant (unchangeable), methods non-overridable, and classes non-inheritable. 
    final int x = 10; // x cannot be changed
  • finally (block): Used in exception handling (try-catch). The finally block always executes, whether an exception occurs or not. It's used for cleanup (e.g., closing a file or database connection). 
    try { ... } 
catch (Exception e) { ... } 
finally { // This always runs }
  • finalize() (method): A method that is called by the garbage collector just before an object is destroyed. It’s used to perform cleanup actions, but it's rarely used and generally discouraged.
  • String:
    • Immutable (cannot be changed).
    • If you modify a String, a new object is created in memory.
    • Use when text doesn’t change much.
  • StringBuilder:
    • Mutable (can be changed).
    • Faster than StringBuffer because it is not thread-safe.
    • Use when working with strings frequently in a single thread.
  • StringBuffer:
    • Mutable (can be changed).
    • Slower than StringBuilder because it is thread-safe (its methods are synchronized).
    • Use when working with strings in multithreaded programs.

An array is a collection of elements of the same data type, stored in a contiguous memory location. Arrays in Java are objects and have a fixed size.

Declaration and Initialization:

// Declaration
int[] arr;
// Initialization (allocating memory)
arr = new int[5]; // array of size 5
// Or, declare and initialize together
int[] arr = {10, 20, 30, 40, 50};

Accessing elements:

System.out.println(arr[0]); // Output: 10 (Arrays are 0-indexed)

Finding length:

System.out.println(arr.length); // Output: 5 (length is a property)

The static keyword means the variable or method belongs to the class, not to an object.

Static Methods:

  • Belong to the class.
  • Can be called without creating an object (e.g., Math.random()).
  • Cannot access non-static (instance) variables directly.
  • Cannot be overridden (this is called method hiding).

Static Variables:

  • Shared by all objects of the class.
  • There is only one copy, no matter how many objects are created.

Access modifiers control the visibility (scope) of classes, methods, and variables.

  • public: Accessible everywhere (from any class, any package).
  • private: Accessible only within the same class.
  • protected: Accessible in the same package and also in subclasses (child classes) in other packages.
  • Default (no modifier): Accessible only within the same package.

Garbage Collection (GC) is the process of automatically freeing memory by destroying objects that are no longer referenced (i.e., have no pointers to them).

This helps prevent memory leaks and means developers don't have to manually free memory (like using free() in C). The garbage collector runs in the background. We can suggest it to run using System.gc(), but it’s not guaranteed.

No, constructors are not inherited in Java. Every class must define its own constructor.

However, a subclass constructor must call the parent class constructor. If you don't explicitly write super(), the compiler will automatically insert a call to the parent's no-argument constructor (super()) as the very first line.

Multithreading allows a program to run multiple threads (small units of execution) simultaneously, improving performance.

There are two ways to create a thread:

  1. Extending the Thread class: 
    class MyThread extends Thread {
    public void run() { ... }
}
MyThread t = new MyThread();
t.start();
  2. Implementing the Runnable interface: (This is preferred because Java does not support multiple inheritance, so you can extend another class if needed). 
    class MyThread implements Runnable {
    public void run() { ... }
}
Thread t = new Thread(new MyThread());
t.start();

The synchronized keyword is used in multithreading to ensure thread safety. It ensures that only one thread can access a specific method or block of code at a time.

This prevents "thread interference" and "data inconsistency" when multiple threads try to modify the same data simultaneously.

class Counter {
    private int count = 0;
    
    // Only one thread can run this method at a time
    public synchronized void increment() {
        count++;
    }
}

Serialization is the process of converting a Java object into a byte stream. This byte stream can then be saved to a file, stored in a database, or sent over a network.

Deserialization is the reverse process: converting the byte stream back into the object.

To make a class serializable, it must implement the Serializable interface (which is a marker interface, meaning it has no methods).

import java.io.Serializable;

class Student implements Serializable {
    String name;
    int rollNo;
}
